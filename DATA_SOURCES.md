# Data Sources and Usage Policy

This document defines how AutoRisk-RM handles input media for evaluation and demos.

## Core policy

- AutoRisk-RM does **not** redistribute third-party source videos.
- Users must verify rights/license before downloading or processing third-party media.
- Public-mode download is opt-in and requires explicit acknowledgement.

## Source registry

| Category | Example path | Origin | License / rights status | Redistributed in repo |
|---|---|---|---|---|
| Public-mode source video | `data/public_samples/uk_dashcam_compilation.mp4` | Third-party URL from `configs/public.yaml` or user `--url` | User must verify rights; default example is YouTube standard license source metadata | No |
| Multi-video local inputs | `data/multi_video/*.mp4` | User-provided local files | User must verify rights | No (`data/multi_video/` ignored) |
| Ground-truth annotations | `data/annotations/*.csv` | Original project annotations | Project-owned metadata | Yes |
| Generated clips/reports | `outputs/**` | Derived analysis artifacts | Generated by user run | No (output directory ignored) |

## Explicit acknowledgement for third-party download

Use:

```bash
python scripts/download_public_data.py --ack-data-rights --config configs/public.yaml
```

Without `--ack-data-rights`, download is rejected.

## Recommended safe workflow

1. Obtain a rights-cleared source video and place it locally.
2. Run pipeline with explicit input path:
   `python -m autorisk.cli -c configs/public.yaml run -i <LOCAL_VIDEO> -o outputs/public_run`
3. For audit-grade deliverables, run:
   `python -m autorisk.cli finalize-run -r outputs/public_run --zip --audit-grade ...`
